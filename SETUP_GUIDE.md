# カレンダー通知ボット：ゼロから始める完全セットアップガイド

プログラミング経験がない方でも、このガイドの手順通りに進めれば必ず動かせます。
所要時間の目安：**約60〜90分**

---

## 全体の流れ

```
STEP 1: GitHubの準備（コードの保管場所）
STEP 2: Googleの準備（カレンダーを読む鍵の作成）
STEP 3: LINEの準備（メッセージを送る窓口の作成）
STEP 4: グループIDの取得（どのグループに送るかの設定）
STEP 5: GitHubに設定を登録
STEP 6: 動作確認
```

---

## STEP 1: GitHub の準備

GitHubはプログラムのコードを保管・自動実行するサービスです。

### 1-1. GitHubアカウントを作成する

1. https://github.com にアクセスする
2. 右上の **「Sign up」** をクリック
3. 以下を順番に入力する：
   - メールアドレス
   - パスワード（8文字以上）
   - ユーザー名（英数字、後から変更可能）
4. 「検証」のパズルを解いて **「Create account」** をクリック
5. 入力したメールに届いた**確認コード**（6桁の数字）を入力する
6. アンケートが表示されたら「Skip personalization」でスキップしてOK

### 1-2. コードを自分のGitHubにコピーする（Fork）

1. https://github.com/ReoReoI/calendar-bot にアクセスする
2. 右上の **「Fork」** ボタンをクリック
3. 設定はそのままで **「Create fork」** をクリック
4. 自分のGitHubアカウントにリポジトリがコピーされる（URLが `github.com/あなたのユーザー名/calendar-bot` になる）

> ✅ ここまでできたらGitHubの準備完了です。一旦このページは開いたままにしておいてください。

---

## STEP 2: Googleの準備

Googleカレンダーのデータを読み取るための「鍵（サービスアカウント）」を作成します。

### 2-1. Google Cloud Consoleにアクセスする

1. https://console.cloud.google.com/ にアクセスする
2. **Googleアカウント**でログインする（通知を受け取りたいGoogleカレンダーと同じアカウント）
3. 初回アクセスの場合は利用規約に同意する

### 2-2. プロジェクトを作成する

1. 画面上部の **「プロジェクトの選択」** をクリック（または「My First Project」と書かれた部分）
2. 右上の **「新しいプロジェクト」** をクリック
3. **プロジェクト名**に `calendar-bot` と入力
4. **「作成」** をクリック
5. 画面上部の通知（ベルアイコン）から `calendar-bot` をクリックして、作成したプロジェクトに切り替える

> ⚠️ 画面上部に「calendar-bot」と表示されていることを確認してから次に進んでください。

### 2-3. Google Calendar APIを有効にする

1. 左上のハンバーガーメニュー（三本線）をクリック
2. **「APIとサービス」** → **「ライブラリ」** をクリック
3. 検索欄に `Google Calendar API` と入力してEnter
4. **「Google Calendar API」** をクリック
5. **「有効にする」** ボタンをクリック
6. 「APIが有効になりました」と表示されればOK

### 2-4. サービスアカウントを作成する

1. 左上のハンバーガーメニュー → **「IAMと管理」** → **「サービスアカウント」** をクリック
2. 上部の **「＋ サービスアカウントを作成」** をクリック
3. **「サービスアカウント名」** に `calendar-bot` と入力
4. **「サービスアカウントID」** は自動入力される（変更不要）
5. **「作成して続行」** をクリック
6. 「このサービスアカウントにプロジェクトへのアクセスを許可する」はスキップ → **「続行」**
7. 「ユーザーにこのサービスアカウントへのアクセスを許可」もスキップ → **「完了」**

### 2-5. サービスアカウントのメールアドレスをコピーする

1. サービスアカウントの一覧に `calendar-bot@calendar-bot-xxxxx.iam.gserviceaccount.com` のようなメールアドレスが表示される
2. このメールアドレスを**メモ帳などにコピーして保存**しておく（後で使います）

### 2-6. JSONキー（鍵ファイル）をダウンロードする

1. 作成したサービスアカウント（`calendar-bot@...`）をクリック
2. 上部の **「キー」** タブをクリック
3. **「鍵を追加」** → **「新しい鍵を作成」** をクリック
4. **「JSON」** を選択して **「作成」** をクリック
5. JSONファイルが自動でダウンロードされる
6. ダウンロードしたファイル（`calendar-bot-xxxxxx.json` のような名前）をデスクトップなど**わかりやすい場所**に保存する

> ⚠️ このJSONファイルは他人に見せないでください。カレンダーへのアクセス権限が含まれています。

### 2-7. GoogleカレンダーをサービスアカウントとShare（共有）する

1. https://calendar.google.com/ にアクセスする
2. 左サイドバーにあるカレンダー名の右側の **「⋮」（三点メニュー）** をクリック
3. **「設定と共有」** をクリック
4. 「特定のユーザーまたはグループと共有する」の右側の **「＋ ユーザーを追加」** をクリック
5. 先ほどコピーしたサービスアカウントのメールアドレス（`calendar-bot@...`）を貼り付ける
6. 権限を **「予定の表示（すべての予定の詳細）」** に設定する
7. **「送信」** をクリック

### 2-8. カレンダーIDをコピーする

1. 同じ「設定と共有」画面の**左サイドバー**でカレンダー名をクリック
2. 画面を下にスクロールして **「カレンダーの統合」** セクションを探す
3. **「カレンダーID」** に表示されている文字列（`xxxxx@gmail.com` や `xxxxx@group.calendar.google.com`）をコピーして**メモ帳に保存**する

> ✅ ここまでできたらGoogleの準備完了です。

---

## STEP 3: LINEの準備

LINEにメッセージを送るための「窓口（チャンネル）」を作成します。

### 3-1. LINE Developersにログインする

1. https://developers.line.biz/ にアクセスする
2. 右上の **「ログイン」** をクリック
3. **LINEアカウント**でログインする（QRコードログインまたはメールアドレス）

### 3-2. プロバイダーを作成する

> プロバイダーとは「このボットを誰が作ったか」を示す箱のようなものです。

1. ログイン後、**「Create a new provider」** をクリック
2. プロバイダー名に任意の名前（例：`自分の名前` や `カレンダーBot`）を入力
3. **「Create」** をクリック

### 3-3. Messaging APIチャンネルを作成する

1. 「Create a new channel」の画面で **「Messaging API」** をクリック
2. 以下を入力する：

   | 項目 | 入力内容 |
   |------|---------|
   | Channel name | `カレンダー通知`（任意） |
   | Channel description | `Googleカレンダーの予定を通知するBot`（任意） |
   | Category | `個人`（任意） |
   | Subcategory | 任意 |

3. 利用規約に同意して **「Create」** をクリック
4. 確認ダイアログも **「OK」** をクリック

### 3-4. チャンネルアクセストークンを発行する

1. 作成したチャンネルの **「Messaging API設定」** タブをクリック
2. 画面を一番下までスクロールする
3. **「チャンネルアクセストークン（長期）」** の右にある **「発行」** ボタンをクリック
4. 表示された長い文字列を**コピーしてメモ帳に保存**する

> ⚠️ このトークンも他人に見せないでください。

### 3-5. ボットをLINEで友達追加する

1. 同じ「Messaging API設定」タブの上部に**QRコード**が表示されている
2. スマートフォンのLINEアプリでQRコードを読み取る
3. 「友達追加」してボットを友達に追加する
4. 通知を受け取りたいグループのメンバー全員も同様に友達追加してもらう

> ✅ ここまでできたらLINEの準備完了です。

---

## STEP 4: LINE グループIDの取得

どのグループに通知を送るかを設定するため、グループIDを取得します。

### 4-1. 一時的なWebhookURLを取得する

1. https://webhook.site にアクセスする（登録不要・無料）
2. 自動で `https://webhook.site/xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx` のようなURLが表示される
3. このURL（上部に表示されている「Your unique URL」）を**コピーしてメモ帳に保存**する
4. **このページは閉じずに開いたままにしておく**

### 4-2. LINE DevelopersにWebhookURLを設定する

1. https://developers.line.biz/ に戻る
2. 作成したチャンネル → **「Messaging API設定」** タブ
3. **「Webhook URL」** の **「編集」** をクリック
4. 先ほどコピーした `https://webhook.site/xxxxxxxx-xxxx-...` を貼り付ける
5. **「更新」** をクリック
6. **「検証」** をクリックして「成功」と表示されることを確認する
7. **「Webhookの利用」** をオン（青色）にする
8. **「グループ・複数人トークへの参加を許可する」** をオン（青色）にする

### 4-3. ボットをLINEグループに招待する

1. スマートフォンのLINEアプリを開く
2. 通知を送りたいグループを開く
3. 右上の**設定アイコン（歯車）** をタップ
4. **「メンバーを招待」** をタップ
5. 先ほど友達追加したボット（「カレンダー通知」など）を選択して招待する

> ボットが「このグループのIDは〇〇です！」と返信してきた場合はSTEP 4-5へ。
> 返信がない場合はSTEP 4-4へ進む。

### 4-4. 「ID教えて」とグループで送信する（招待済みの場合）

すでにボットがグループにいる場合：
1. グループのトーク画面を開く
2. **「ID教えて」** とメッセージを送信する

### 4-5. webhook.siteでグループIDを確認する

1. 開いたままにしていた webhook.site のページを確認する
2. 左側に受信したデータが表示されている（表示されない場合はページを更新）
3. 右側のJSONデータの中から `"groupId"` という項目を探す：
   ```
   "source": {
       "type": "group",
       "groupId": "Cxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
   }
   ```
4. `"groupId"` の値（`C` から始まる長い文字列）を**コピーしてメモ帳に保存**する

### 4-6. WebhookURLを削除する（後片付け）

グループIDが取得できたので、一時的なWebhookは不要になります。

1. LINE Developers → 「Messaging API設定」タブ
2. **「Webhookの利用」** をオフにする
3. Webhook URL を削除または `https://webhook.site` のみにしておく

### 4-7. LINE Official Account ManagerでBot設定を変更する

グループ内でメッセージを送るたびにボットが自動返信しないように設定します。

1. https://manager.line.biz/ にアクセスする
2. 作成したチャンネル（「カレンダー通知」など）をクリック
3. 左メニューの **「設定」** → **「応答設定」** をクリック
4. 以下をすべて**オフ**にする：
   - **「あいさつメッセージ」** → オフ
   - **「応答メッセージ」** → オフ

> ✅ ここまでできたらLINEの設定完了です。

---

## STEP 5: GitHubに設定を登録する

集めた情報をGitHubの「金庫（Secrets）」に登録します。

### 5-1. JSONファイルの中身を確認する

1. STEP 2-6でダウンロードしたJSONファイル（`calendar-bot-xxxxxx.json`）を**メモ帳**で開く
   - ファイルを右クリック → 「プログラムから開く」→「メモ帳」
2. 表示された内容（`{` から始まり `}` で終わる長い文字列）を**すべて選択してコピー**する

### 5-2. GitHub SecretsにSecretsを登録する

1. https://github.com/あなたのユーザー名/calendar-bot にアクセスする
2. 上部のタブから **「Settings」** をクリック
3. 左メニューの **「Secrets and variables」** → **「Actions」** をクリック
4. **「New repository secret」** をクリックして、以下の4つを1つずつ登録する：

---

**① GOOGLE_SERVICE_ACCOUNT_JSON**

- Name: `GOOGLE_SERVICE_ACCOUNT_JSON`
- Secret: JSONファイルの中身をすべて貼り付ける
- **「Add secret」** をクリック

---

**② CALENDAR_ID**

- Name: `CALENDAR_ID`
- Secret: STEP 2-8でコピーしたカレンダーID（例：`yourname@gmail.com`）
- **「Add secret」** をクリック

---

**③ LINE_CHANNEL_ACCESS_TOKEN**

- Name: `LINE_CHANNEL_ACCESS_TOKEN`
- Secret: STEP 3-4でコピーしたアクセストークン
- **「Add secret」** をクリック

---

**④ LINE_GROUP_ID**

- Name: `LINE_GROUP_ID`
- Secret: STEP 4-5でコピーしたグループID（`C` から始まる文字列）
- **「Add secret」** をクリック

---

4つ登録できたら、Secrets一覧に以下が表示されていることを確認する：

```
CALENDAR_ID
GOOGLE_SERVICE_ACCOUNT_JSON
LINE_CHANNEL_ACCESS_TOKEN
LINE_GROUP_ID
```

---

## STEP 6: 動作確認

実際に通知が届くかテストします。

### 6-1. テスト用の予定をカレンダーに追加する（任意）

1. Googleカレンダーに**今日の予定**をいくつか追加しておく
2. 予定があると通知内容が確認しやすい

### 6-2. 手動でボットを実行する

1. https://github.com/あなたのユーザー名/calendar-bot にアクセスする
2. 上部のタブから **「Actions」** をクリック
3. 左側の **「朝のLINE通知（8:00 JST）」** をクリック
4. 右側の **「Run workflow」** ボタンをクリック
5. 表示されたドロップダウンの **「Run workflow」** をクリック
6. ページを更新すると実行中のワークフローが表示される
7. 緑のチェックマーク ✅ が表示されれば成功

### 6-3. LINEグループで通知を確認する

LINEグループに以下のような通知が届いていれば完成です！

```
📅 本日のスケジュール（○月○日）

① 10:00 〜 11:00　ミーティング
② 14:00 〜 15:00　打ち合わせ
```

予定がない日は「今日は予定がありません」と届きます。

---

## 自動実行について

毎朝 **8:00（日本時間）** に自動で実行されます。設定変更は不要です。

---

## うまくいかない場合のチェックリスト

| 症状 | 確認すること |
|------|------------|
| Actions が赤い ❌ | Secretsの名前や値にタイポがないか確認 |
| LINEに通知が届かない | LINE_GROUP_IDが正しいか / ボットがグループにいるか確認 |
| カレンダーが取得できない | サービスアカウントのメールをカレンダーと共有したか確認 |
| ボットが自動返信する | LINE Official Account Manager の応答メッセージをオフにする |

## まとめ：メモしておく情報

セットアップ中に控えた情報は安全な場所に保管してください。

| 情報 | 保管場所 |
|------|---------|
| JSONファイル | パソコン内の安全な場所（クラウドにアップしない） |
| チャンネルアクセストークン | LINE Developers Console から再発行可能 |
| グループID | GitHub Secrets に登録済み |
